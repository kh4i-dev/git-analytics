Git Diff Workflow

Để AI gợi ý commit và review chính xác nhất, bạn cần cung cấp mã diff tiêu chuẩn từ terminal của dự án:

  • Chưa stage (chưa git add): git diff
  • Đã stage (đã chạy git add): git diff --cached
  • Commit gần nhất: git show HEAD

Conventional Commits

AI sẽ phân tích code thay đổi và tự động viết commit message theo quy chuẩn chuẩn hóa của các team công nghệ toàn cầu:

<type>(<scope>): <description>
  • feat: Tính năng mới (ví dụ: feat(auth):...)
  • fix: Sửa lỗi hệ thống
  • refactor: Cấu trúc lại code
  • docs: Cập nhật tài liệu

Repo Assistant

Hỏi trợ lý AI về cấu trúc mã nguồn, quy trình nghiệp vụ dựa trên codebase đã được đồng bộ hóa của bạn:

  • "Kiến trúc đồng bộ repository (Sync Service) hoạt động như thế nào?"
  • "Quy trình auth và đăng nhập bằng GitHub được thiết kế ra sao?"
  • "Đề xuất bộ test case cho chức năng chuyển đổi cấu hình BYOK."
BYOK Security: Khóa BYOK của bạn được mã hóa an toàn bằng thuật toán Fernet đối xứng trước khi lưu vào DB. Tài liệu kỹ thuật đầy đủ
